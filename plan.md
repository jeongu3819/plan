## 로드맵(간트 차트) UI 요구사항 (월/주/일 + 하위 프로젝트)

아래 이미지는 목표로 하는 **간트 차트(로드맵) 화면 예시**입니다.

![로드맵(간트 차트) 예시](https://raw.githubusercontent.com/jeongu3819/plan/main/chart.png)

### 핵심 목표
- **월별 / 주별 / 일별**로 동일 데이터를 서로 다른 뷰로 전환해서 볼 수 있어야 함
- **하위 프로젝트(서브프로젝트)**까지 같은 로드맵에 자연스럽게 포함되어야 함
- 아래 필드를 기준으로 **깔끔하고 한눈에 들어오게** 표시되어야 함

### 로드맵에 반드시 보여야 하는 기본 컬럼/필드
- **업무명(또는 프로젝트/Task명)**
- **시작일(Start date)**
- **마감일/종료일(Due/End date)**
- **상태(Status)**: 예) Not Started / In Progress / Blocked / Done (표현은 단순·일관되게)
- **담당자(Owner/Assignee)** (필요 시)
- **진행률(%)** (필요 시)

### 하위 프로젝트 표현 방식(권장)
- 프로젝트(상위) 행 아래에 **하위 프로젝트/Task를 들여쓰기(Indent)** 또는 **트리(접기/펼치기)** 형태로 표시
- 상위 프로젝트는 하위 항목들의 진행률/일정을 **롤업(합산/범위 표시)** 해서 한 줄로도 보이게

### 한눈에 들어오게 만드는 UX 요소(권장)
- 뷰 전환: **Month / Week / Day** 탭 또는 토글
- 필터/정렬: 담당자, 상태, 기간(이번주/이번달), 지연(Overdue) 필터
- 오늘 기준선(Vertical line) + 마감일 지난 항목은 **지연 표시(Overdue)**
- 항목 클릭 시: 상세(설명/링크/첨부/메모) 패널 또는 모달로 연결
- 리스트/보드(칸반)와 **같은 데이터 소스**를 쓰고, 표현만 다르게(로드맵/칸반/리스트/캘린더) 구성
